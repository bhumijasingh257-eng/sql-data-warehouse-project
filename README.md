
# Data Warehouse Project

## Overview
This project demonstrates the design and implementation of a Data Warehouse for storing, integrating, and analyzing data from multiple sources. The warehouse is built to support business intelligence, reporting, and data analytics.

## Objectives
- Centralize data from different sources
- Improve data quality and consistency
- Enable efficient reporting and analysis
- Support decision-making through historical data

## Architecture
1. Data Sources
2. ETL Process (Extract, Transform, Load)
3. Data Warehouse
4. Reporting & Analytics

## Technologies Used
- SQL
- MySQL
- ETL Tools
- Git & GitHub

## Database Schema
The warehouse follows a Star Schema design:
- Fact Table: Sales_Fact
- Dimension Tables:
  - Customer_Dim
  - Product_Dim
  - Time_Dim
  - Location_Dim

## ETL Process
1. Extract data from source systems
2. Clean and transform data
3. Load data into dimension tables
4. Load data into fact tables

## Features
- Historical data storage
- Data aggregation
- Business reporting
- Performance optimization
